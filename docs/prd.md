# CrewUp — 요구사항 정의서 (PRD)

> 이 문서는 CrewUp 플랫폼의 전체 요구사항을 정의합니다.
> MVP(Phase 0~5)부터 장기 비전(Phase 11)까지의 기능, 비즈니스 로직, 데이터 모델을 상세히 기술합니다.
> 기술 스택과 프로젝트 구조는 `/CLAUDE.md`에 정의되어 있습니다.

---

## 1. 제품 개요

### 1.1 한 줄 요약
동호회 운영자가 동호회를 만들고 체계적으로 관리하며, 일반인이 관심 있는 동호회를 찾아 가입할 수 있는 플랫폼.

### 1.2 풀고자 하는 문제

| 사용자 | Pain Point |
|--------|-----------|
| 동호회 운영자 | 카톡 단톡방으로 일정/참석/회비 관리가 비효율적. 누가 참석하는지, 돈을 냈는지 매번 확인해야 함 |
| 동호회 회원 | 일정이 언제인지, 몇 명 참석하는지, 참가비가 얼마인지 단톡방을 뒤져야 함 |
| 일반인 | 가입하고 싶은 동호회를 찾기 어렵고, 가입 절차가 불투명 (지인 소개에 의존) |

### 1.3 타겟 사용자

| 사용자 유형 | 역할 | 주요 니즈 |
|------------|------|----------|
| **클럽장 (Owner)** | 동호회를 만들고 전반을 관리 | 일정/멤버/회비 관리, 공지, 통계, 운영진 지정 |
| **운영진 (Admin)** | 클럽장이 지정한 보조 관리자 | 일정 생성, 멤버 승인/거절, 공지 작성 |
| **일반 멤버 (Member)** | 동호회에 가입한 회원 | 일정 확인, 참석/대기, 알림 수신, 통계 조회 |
| **비회원 (Guest)** | 가입 전 탐색하는 일반인 | 동호회 검색, 상세 보기, 가입 신청 |

### 1.4 성장 전략
1. 인스타그램/SNS에서 동호회 링크 공유 → 자연 유입
2. 가입 시 플랫폼 접속 필수 → 비회원이 회원으로 전환
3. 동호회 운영 편의성으로 운영자 → 회원 → 새 동호회 확산
4. 동호회 내 알림이 핵심 — 알림 받으려면 앱 사용 필수

### 1.5 경쟁 분석

| 서비스 | 특징 | CrewUp 차별점 |
|--------|------|---------------|
| 네이버 카페 | 범용 커뮤니티, 일정 관리 기능 약함 | 동호회 특화 일정/참석/알림 시스템 |
| 소모임 | 모임 찾기 중심, 운영 도구 부족 | 클럽장 관리 도구 + 참석/대기 자동화 |
| 플레이팅 | 스포츠 매칭 중심, 동호회 관리 X | 동호회 운영 + 개인 매칭 모두 지원 (Phase 11) |
| 카카오톡 단톡방 | 현재 대부분의 동호회가 사용 | 일정/참석/회비를 체계적으로 관리, 알림 자동화 |

### 1.6 성공 지표 (KPI)

**MVP 성공 기준 (출시 후 3개월):**

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 등록 동호회 수 | 50개 | DB 집계 |
| 총 가입 회원 수 | 1,500명 (동호회당 평균 30명) | DB 집계 |
| 주간 활성 사용자 (WAU) | 500명 | 주 1회 이상 접속 |
| 월간 일정 생성 수 | 200건 | DB 집계 |
| 평균 참석률 | 70% 이상 | 참석/정원 비율 |
| 알림 → 앱 접속 전환율 | 30% 이상 | 푸시 클릭률 |

### 1.7 디바이스 전략 & UI 원칙

> 대부분의 일반 멤버는 **모바일**로 접속하고, 클럽장/운영진은 관리 작업 시 **데스크톱**도 사용한다.
> 이를 반영하여 모바일 퍼스트로 설계하되, 데스크톱에서도 관리가 편하도록 반응형을 적용한다.

**사용자별 주요 디바이스:**

| 사용자 | 주요 디바이스 | 사용 시나리오 |
|--------|-------------|-------------|
| 일반 멤버 | 모바일 (90%+) | 알림 확인, 참석 신청, 일정 조회 — 빠르고 간편하게 |
| 클럽장/운영진 | 모바일 (60%) + 데스크톱 (40%) | 일정 생성, 멤버 관리, 통계 — 모바일에서도 가능하지만 데스크톱이 편함 |
| 비회원 | 모바일 (80%+) | SNS 링크 → 동호회 탐색 → 가입 신청 |

**UI 설계 원칙:** (상세 디자인 규격은 `/DESIGN.md` 참조)
- **모바일 퍼스트**: 모든 화면을 모바일에서 먼저 완성한 후 데스크톱으로 확장
- **반응형 필수**: 같은 코드로 모바일/태블릿/데스크톱 대응 (NativeWind 반응형 클래스)
- **모바일 핵심 동선 최소화**: 알림 탭 → 일정 상세 → 참석 버튼 = 3탭 이내
- **데스크톱 관리 효율**: 관리 페이지는 데스크톱에서 사이드바 + 넓은 테이블 레이아웃
- **네비게이션**: 모바일은 하단 탭 바, 데스크톱은 상단 GNB 또는 사이드바
- **카드 그리드**: 모바일 1열 → 태블릿 2열 → 데스크톱 3열
- **모달/입력**: 모바일은 바텀시트, 데스크톱은 센터 모달

## 2. 사용자 플로우

### 2.1 최초 가입/로그인

```
1. 앱/웹 접속
2. "카카오로 시작하기" 또는 "네이버로 시작하기" 탭
3. 소셜 로그인 진행
4. [최초 가입 시] 온보딩 화면:
   - 닉네임 설정 (필수, 중복 체크)
   - 프로필 사진 (선택)
   - 활동 지역 (선택 — 동호회 추천에 활용)
5. 메인 화면 (동호회 탐색) 진입
```

### 2.2 동호회 탐색 → 가입

```
1. 메인 탭 "홈"에서 동호회 목록 탐색
2. 검색 (이름, 지역) 또는 필터 (종목, 지역, 모집중) 적용
3. 관심 동호회 카드 탭 → 동호회 소개 페이지
4. 소개, 활동 지역, 멤버 수, 모집 여부 확인
5. "가입 신청" 버튼 탭
6. [미로그인 시] 로그인 화면으로 이동 → 로그인 후 복귀
7. 가입 인사말 작성 (선택) → "신청하기" 탭
8. "가입 대기 중" 안내 → 알림 대기
9. 클럽장/운영진이 승인 → 푸시 알림 수신
10. "내 동호회" 탭에서 가입된 동호회 확인
11. 동호회 내부 (일정, 멤버, 통계) 접근 가능
```

### 2.3 동호회 생성 (클럽장)

```
1. "내 동호회" 탭 → "동호회 만들기" 탭
2. 동호회 정보 입력:
   - 이름, 설명
   - 종목 선택 (MVP: 테니스)
   - 활동 지역
   - 모임 유형 (정기/번개)
   - 최대 멤버 수
   - 공개/비공개
   - 대표 이미지 (선택)
3. "만들기" 탭 → 동호회 생성 완료
4. 본인이 자동으로 클럽장(Owner)으로 등록
5. 동호회 관리 페이지 접근 가능
6. 링크 공유하여 멤버 모집 시작
```

### 2.4 일정 생성 (운영진)

```
1. 동호회 내부 → "일정관리" 탭
2. "일정 생성" 버튼 탭
3. 일정 정보 입력:
   - 날짜, 시작/종료 시간
   - 장소 (자주 쓰는 장소 선택 또는 직접 입력)
   - 정원 (2~20명)
   - 참가비
   - 메모 (안내 사항)
   - 옵션: 등록자 자동 참석, 대기자 허용, 전체 알림
4. 미리보기 확인 → "생성" 탭
5. [전체 알림 ON 시] 동호회 전체 멤버에게 푸시 알림 발송
6. 캘린더에 일정 카드 표시
```

### 2.5 일정 참석/대기 (멤버)

```
1. 푸시 알림 "새 일정이 등록되었습니다" 수신 → 탭하여 앱 진입
   또는 동호회 캘린더에서 일정 카드 탭
2. 일정 상세 확인 (날짜, 장소, 참가비, 현재 참석자)
3. "참석" 버튼 탭
4. [정원 미달] → 즉시 참석 확정, "참석중" 뱃지 표시
   [정원 초과] → 대기자로 등록, "대기 N번" 표시
5. 참석 취소: "참석 취소" 버튼 → 확인 팝업 → 취소 완료
6. [대기자 있는 상태에서 참석자 취소 시]
   → 대기 1번이 자동으로 참석 승격
   → 승격된 사용자에게 푸시 알림 "참석이 확정되었습니다"
```

### 2.6 알림 → 행동 플로우

```
1. 푸시 알림 수신 (새 일정, 변경, 취소, 가입 승인 등)
2. 알림 탭 → 앱 내 해당 화면으로 딥링크 이동
   - "새 일정" 알림 → 일정 상세 화면
   - "가입 승인" 알림 → 동호회 메인 화면
   - "가입 신청" 알림 → 가입 대기 관리 화면
3. 앱 내 알림 탭에서 알림 목록 확인 (읽음/안읽음)
```

### 2.7 동호회 관리 (클럽장/운영진)

```
1. 동호회 내부 → "관리" 탭 (클럽장/운영진에게만 표시)
2. 기능:
   - 동호회 정보 수정
   - 가입 신청 목록 → 승인/거절
   - 멤버 목록 → 역할 변경 / 강퇴
   - 운영진 지정/해제 (클럽장만)
   - 자주 쓰는 장소 관리
   - 공지사항 작성
3. 가입 신청 처리 → 신청자에게 알림 발송
4. 강퇴 → 해당 멤버에게 알림 발송 + 동호회 접근 차단
```

## 3. 기능 상세 & 우선순위 (MVP)

### 우선순위 분류

- **M (Must-have)**: 이거 없으면 MVP가 아님
- **S (Should-have)**: 있으면 좋지만 없어도 출시 가능
- **N (Nice-to-have)**: 여유 있으면 추가

### 3.1 인증 [M]

| 기능 | 우선순위 | 비즈니스 규칙 |
|------|---------|-------------|
| 카카오 소셜 로그인 | M | Supabase Auth OAuth 연동 |
| 네이버 소셜 로그인 | M | Supabase Auth OAuth 연동 |
| 온보딩 (닉네임 설정) | M | 닉네임 필수, 2~12자, 중복 불가 |
| 프로필 사진 | S | Supabase Storage 업로드, 최대 5MB |
| 활동 지역 선택 | S | 시/도 + 시/군/구, 동호회 추천에 활용 |
| 자동 세션 유지 | M | 토큰 자동 갱신, 앱 복귀 시 유효성 확인 |
| 로그아웃 | M | 세션 삭제 + 로컬 캐시 초기화 |

### 3.2 동호회 탐색 & 가입 [M]

| 기능 | 우선순위 | 비즈니스 규칙 |
|------|---------|-------------|
| 동호회 목록 | M | 카드 형태, 비회원도 접근 가능 |
| 검색 (이름) | M | 부분 일치 검색 |
| 필터 (지역, 종목, 모집중) | S | 다중 필터 조합 가능 |
| 동호회 소개 페이지 | M | 비회원 접근 가능, 내부 콘텐츠는 잠금 |
| 가입 신청 | M | 로그인 필수, 인사말 선택 |
| 가입 대기 상태 표시 | M | "대기 중" / "승인됨" / "거절됨" |

### 3.3 동호회 생성 & 관리 [M]

| 기능 | 우선순위 | 비즈니스 규칙 |
|------|---------|-------------|
| 동호회 생성 | M | 생성자가 자동으로 Owner |
| 동호회 정보 수정 | M | Owner/Admin만 |
| 운영진 지정/해제 | M | Owner만 |
| 가입 승인/거절 | M | Owner/Admin |
| 멤버 강퇴 | M | Owner/Admin, 본인 강퇴 불가 |
| 동호회 삭제 | S | Owner만, 확인 팝업 필수 |
| 자주 쓰는 장소 관리 | S | Owner/Admin, 일정 생성 시 선택 가능 |

### 3.4 멤버 관리 [M]

| 기능 | 우선순위 | 비즈니스 규칙 |
|------|---------|-------------|
| 멤버 목록 | M | 역할, 가입일, 참석률, 참석 횟수 표시 |
| 멤버 검색/필터 | S | 이름 검색, 역할별 필터 |
| 역할 변경 | M | Owner만, Owner→Admin/Admin→Member |
| 내 동호회 목록 | M | 가입된 모든 동호회 표시, 전환 가능 |
| 멤버 통계 바 | S | 전체/운영진/회원 수 + 평균 참석률 |

### 3.5 일정 관리 [M]

| 기능 | 우선순위 | 비즈니스 규칙 |
|------|---------|-------------|
| 캘린더 뷰 (월별) | M | 월 이동, 오늘 강조, 지난 일정 흐리게 |
| 일정 생성 | M | Owner/Admin, 필수: 날짜/시간/장소/정원/참가비 |
| 일정 수정 | M | Owner/Admin, 변경 시 참석자에게 알림 |
| 일정 취소 | M | Owner/Admin, 취소 사유 입력, 참석자에게 알림 |
| 일정 상세 | M | 참석자/대기자 명단, 장소, 참가비 |
| 참석 신청 | M | 정원 미달 시 즉시 확정, 초과 시 대기 |
| 대기자 자동 승격 | M | 참석자 취소 시 대기 1번 자동 승격 + 알림 |
| 참석 취소 | M | 일정 시작 전까지 가능 |
| 참가비 입금 안내 | S | 계좌 정보 표시 + 복사 (Phase 9 결제 전까지) |

**일정 비즈니스 규칙 상세:**
- 정원: 2~20명 (운영진이 설정)
- 정원 마감: 참석자 수 = 정원 → 자동 마감 (status: closed)
- 대기자 허용: 옵션 ON 시 정원 초과 대기 가능
- 대기 → 참석 승격: 선착순 (position 기준), 승격 시 푸시 알림
- 참석 취소 후 재참석: 가능, 새로운 position으로 대기열 진입
- 일정 시작 이후: 참석/취소 버튼 비활성화

### 3.6 알림 [M]

| 알림 유형 | 우선순위 | 트리거 | 수신 대상 | 딥링크 |
|----------|---------|--------|----------|--------|
| 새 일정 | M | 일정 생성 (전체 알림 ON) | 동호회 전체 멤버 | 일정 상세 |

| 대기→참석 승격 | M | 참석자 취소로 자리 발생 | 대기 1번 | 일정 상세 |

**알림 방식:**
- MVP: FCM 푸시 + 앱 내 알림 목록 (읽음/안읽음)
- 향후: 카카오 알림톡 (사업자 등록 후)

## 5. 엣지 케이스 & 예외 처리

| 상황 | 처리 방법 |
|------|----------|
| **클럽장 탈퇴** | 탈퇴 전 클럽장 이전 필수 (Admin에게). 이전 안 하면 탈퇴 불가 |
| **클럽장이 유일한 멤버인 채 탈퇴** | 동호회 자동 삭제 (확인 팝업) |
| **동호회 삭제** | Owner만, 전체 멤버에게 알림 → 관련 데이터 soft delete |
| **계정 삭제** | 프로필 익명화, 동호회 탈퇴 처리, 클럽장이면 이전 필수 |
| **동시 참석 신청 (race condition)** | DB 트랜잭션 + 유니크 제약조건으로 방지, 정원 초과 시 대기로 자동 전환 |
| **대기자 승격 중 취소** | 트랜잭션 내에서 처리, 승격 실패 시 다음 대기자로 |
| **네트워크 오프라인** | 앱 내 오프라인 안내, 온라인 복구 시 자동 동기화 (react-query) |
| **푸시 토큰 만료** | 앱 시작 시 토큰 갱신, 실패 시 재등록 |
| **이미 가입된 동호회에 재신청** | 가입 신청 버튼 비활성화, "이미 멤버입니다" 안내 |
| **거절된 동호회에 재신청** | 거절 후 7일 이후 재신청 가능 |
| **일정 시작 이후 참석/취소** | 버튼 비활성화, "이미 시작된 일정입니다" 안내 |

**법적 고려사항:**
- 사업자 등록

- 결제 관련 소비자 보호법

- 종목별 특화 필드 (골프 핸디캡, 축구 포지션 등)
- 지역 기반 동호회 추천, 지역별 랭킹
- 목적별 분류 (정기/번개/리그/레슨), 레벨별 필터
- **의존성**: MVP의 종목 무관 데이터 구조 활용

### Phase 11 — 지역 기반 + 커뮤니티

**핵심 가치**: 동호회 외 개인 유저까지 품기
- 개인 유저 코트/장소 올리기 + 참여자 모집 (인력시장 300~400명)
- 실력/지역/시간대 기반 매칭
- 커뮤니티 (게시판, 장비 거래, 코트 후기)
- **의존성**: Phase 9 결제 + Phase 10 다종목

## 7. 비기능적 요구사항

### 성능
- 앱 초기 로딩: 3초 이내
- 화면 전환: 300ms 이내
- 리스트 스크롤: 60fps
- 알림 지연: 5초 이내
- 동시 접속: 초기 1,500명 → 확장 10,000명+

### 확장성
- 데이터 모델: 종목/지역/목적에 종속되지 않는 범용 구조
- 종목별 특화 필드는 JSONB 또는 별도 테이블로 확장
- 테이블명/컬럼명에 특정 종목 하드코딩 금지

### 보안
- 소셜 로그인 + 세션 관리 (Supabase Auth)
- RLS로 데이터 접근 제어 (회원만 동호회 내부 접근)
- service_role 키 클라이언트 노출 금지
- 개인정보 암호화
- 결제 정보 PCI DSS 준수 (Phase 9)

### 접근성
- 터치 영역 최소 44x44pt
- 색상만으로 정보 전달하지 않기
- accessibilityLabel 설정 (스크린 리더 대응)

### 플랫폼 지원
- MVP: 웹 (PWA) — 모바일 브라우저 최적화
- 확장: Android (Google Play) → iOS (App Store)
- 반응형: 모바일 우선, 태블릿/데스크톱 대응
