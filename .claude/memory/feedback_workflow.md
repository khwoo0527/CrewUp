---
name: 작업 진행 방식
description: 코드 변경 시 반드시 확인 요청 후 진행, 커밋/푸쉬는 빌드+실행 확인 후에만 제안
type: feedback
---

- 코드 변경/파일 생성 전에 반드시 내용을 보여주고 확인 받은 후 진행
- 대규모 수정 후에는 사용자에게 확인 요청
- git 커밋/푸쉬는 빌드가 성공하고 실행까지 문제없는 상태가 확인된 후에만 제안
- 절대 사용자 확인 없이 git 명령을 실행하지 않는다
- 차근차근 단계별로 진행하고, 한꺼번에 쏟아내지 않는다
- 여러 파일 수정 시 1개씩 보여주고 검토 받은 후 다음 파일 진행 (연관된 파일만 예외적으로 같이)
- 대량 작업(8개 파일 동시 등) 절대 금지 — 검토하기 힘듦
- 문서/코드 작성 시 항상: 상세하고, 꼼꼼하고, 있어보이게, 베스트 프랙티스로 작성
- 바이브 코딩 시 더 좋은 결과가 나오도록 고민하며 작성 (매번 말하지 않아도 기본 원칙)
